# AI Native 效能看板设计说明（V1.0）

## 1. 背景与目标
- 背景：AI 开发工具快速普及，但缺少统一口径衡量“是否转型成功”。
- 目标：建立可解释、可比较、可行动的 AI Native 效能度量体系，覆盖采用面、产出、效率与质量。

## 2. 设计原则
- 逻辑闭环：从“采用”到“产出”再到“效率/质量”。
- 口径一致：统一时间窗口与定义，降低解释成本。
- 可行动：指标异常可定位原因并推动改善。
- 风险可见：对潜在误读场景明确提示。

## 3. 看板层级与结构
- 体系级：总体转型进度与健康度（战略层）
- 产线级：不同产线的采用与效能差异（管理层）
- 项目级：周期与指标落地情况（执行层）

### 图示结构（指标逻辑链）
```
采用面（人员覆盖率） ──> 产出量（硅基含量） ──> 效率（节省人日）
                                    │
                                    └────────> 质量（自动化首通率）
```

### 图示结构（看板层级）
```
体系级看板
  ├─ 产线级看板（每条产线）
  │    ├─ 项目级明细
  │    └─ 关键指标对齐展示
```

## 4. 指标体系定义

### 4.1 人员覆盖率（Adoption）
- 目的：衡量采用“深度”与“覆盖面”。
- 口径：近30天（按 PR 合并时间，北京时间）。
- 公式：
  - 覆盖人员数 / 近30天有 PR 的人员数
  - 覆盖人员满足：个人 PR 中 AI 代码占比≥80% 的 PR 占比≥80%

### 4.2 硅基含量（Code Coverage）
- 目的：衡量 AI 参与产出的体量贡献。
- 口径：近30天（按 PR 合并时间，北京时间）。
- 公式：
  - AI 代码行数 / 总代码行数

### 4.3 节省人日（估算）
- 目的：衡量 AI 模式带来的效率收益（产能释放）。
- 口径：累计所有在研项目，包含已完成与进行中阶段；未开始阶段不计入。
- 公式：
  - Σ max(0, 计划基线天数 − AI 模式下实际/预测天数) × 阶段平均投入人力
- 备注：表示“可释放产能”，不等于可缩减编制。

### 4.4 自动化案例指标（质量与执行）
- 时间口径统一：近30天（按执行时间，北京时间）。
- 三个子指标：
  1) 自动化案例占比（覆盖面）
     - 活跃自动化用例数 / 活跃全部用例数
  2) AI 自动化执行占比（使用强度）
     - AI 自动化执行次数 / 自动化执行总次数
  3) AI 自动化首通率（质量稳定性）
     - AI 用例首跑通过数 / 首跑总数（按用例）

## 5. 指标设计考虑与理由
- 采用面：避免“少数人高强度使用”误导整体转型进度。
- 产出量：行数加权更能反映真实体量贡献。
- 效率：用基线与实际/预测周期对比，体现周期压缩。
- 质量：以首通率衡量稳定性，避免高频执行把指标“冲高”。

## 6. 风险与误读提示
- 人员覆盖率：低 PR 量人员会引入波动。
- 硅基含量：重复/机械生成可能抬高比例。
- 节省人日：受基线口径与预测误差影响，非编制缩减。
- 自动化执行占比：历史存量非 AI 用例可能稀释执行占比。

## 7. 口径统一说明
- 时间窗口：近30天。
- 时区：北京时间。
- 统计事件：
  - 覆盖率与硅基含量：PR 合并时间。
  - 自动化指标：用例执行时间。

## 8. 使用建议
- 用于趋势判断：看 30 天内提升方向与结构变化。
- 用于诊断：结合产线/项目下钻排查异常。
- 用于沟通：明确“效能指标”与“编制调整”区分。
